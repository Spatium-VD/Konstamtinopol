Новый статус в панели «Объезды»
=============================

В конфиг добавлен статус **«Доставлено в офис»**: промежуточное состояние, когда договор/бумага уже доехали до офиса или внутреннего контура, но ещё не в финальном состоянии «у сотрудника на точке» (как в вашем учёте код **1**).

Соответствие ваших пометок → статус в системе (при импорте CSV):

  1?  → В работе (заказ/доставка в процессе)
  1   → Доставлено в офис
  2   → Доставлено (итог «всё в порядке»)
  0   → Новая
  уехала → Отменено

Файлы
-----

- obyezdy_migracija_source.tsv — исходная таблица (как у вас).
- obyezdy_migracija_google.csv — готово для вставки в лист **Объезды** (15 колонок как в Apps Script).
- build_obyezdy_migracija_csv.py — пересборка CSV после правок TSV.

Импорт в Google Таблицу
-----------------------

1. Откройте лист «Объезды» (строка заголовков уже есть от createDetoursSheet / ensure).
2. Можно **Файл → Импорт → Загрузка** на отдельный временный лист, затем скопировать только **строки данных** (без заголовка) под строку 1 **или** вставить, если лист пустой кроме шапки — тогда вставьте начиная со строки 2 и **включите строку заголовков из CSV** только если совпадает с шапкой листа.
3. ID в файле: **M1…M54**, чтобы не пересекаться с новыми заявками **D1…** из приложения.

После импорта при необходимости в Apps Script можно не трогать — строки читаются как обычные заявки.
