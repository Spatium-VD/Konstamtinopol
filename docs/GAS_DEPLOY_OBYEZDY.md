# Деплой Google Apps Script после раздела «Объезды»

## Что сделано в коде

1. Файл проекта **[вне_хостинга/appscript.js](/Users/volurg/Desktop/Crown_Pit/rome-main-v02/вне_хостинга/appscript.js)** расширен:
   - лист **`Объезды`** v2 (13 колонок, см. `DETOURS_HEADERS` в скрипте);
   - в основной ответ `doGet` без параметров добавлено поле **`detours`** — массив заявок (чтение по заголовкам строки 1, совместимо со старым 15-колоночным листом);
   - поддерживаются действия **`detourCreate`** и **`detourUpdate`** через GET-параметры `action` и `p` (JSON в base64);
   - опционально **`doPost`** с телом `{"action":"detourCreate|detourUpdate","payload":{...}}` для длинных данных.

2. Фронтенд отправляет заявки методом **GET** на тот же URL, что и основная загрузка данных (`CONFIG.appsScriptUrl`), с query:
   - `?action=detourCreate&p=<base64(UTF-8 JSON)>`
   - `?action=detourUpdate&p=<base64(UTF-8 JSON)>`

## Что нужно сделать в Google

1. Открыть **Apps Script**, привязанный к вашей таблице (или тот же проект, что уже деплоится как веб-приложение).
2. **Заменить код** на актуальное содержимое файла `вне_хостинга/appscript.js` из репозитория.
3. Сохранить (**Ctrl+S / Cmd+S**).
4. **Новый деплой** → **Веб-приложение**:
   - выполнять от имени: как у вас уже настроено (обычно «Я»);
   - доступ: **Все, у кого есть ссылка** (как для текущей панели).
5. Скопировать **новый URL** вида `https://script.google.com/macros/s/.../exec` и при необходимости обновить **`appsScriptUrl`** в [config.js](/Users/volurg/Desktop/Crown_Pit/rome-main-v02/config.js) (если при смене деплоя URL изменился).

### Создать лист «Объезды» вручную (рекомендуется один раз)

В редакторе Apps Script после вставки кода:

1. В списке функций выберите **`createDetoursSheet`**.
2. Нажмите **Выполнить** (Run). Разрешите доступ к таблице, если спросит.
3. Лист **`Объезды`** (v2): **13 колонок A–M** — без телефона, типа договора и «желаемой доставки»; есть колонка **«Дата доставки договора»** (в API — `contractDeliveryDate`) для формул и ручного ввода в таблице. Старые листы с 15 колонками скрипт по-прежнему **читает** по строке заголовков; новые заявки из приложения добавляются в соответствии с текущими заголовками листа.

Альтернатива: ничего не запускать — лист всё равно создастся автоматически при первом запросе к API или первой заявке из приложения.

После деплоя при первом обращении к данным или при первой заявке лист **«Объезды»** также может быть создан автоматически функцией `ensureDetoursSheet_`.

## Если заявка не создаётся

- Проверить, что открыта именно **та же** таблица, что в скрипте (`SPREADSHEET_ID` в начале `appscript.js`).
- Проверить в браузере консоль сети: не обрезается ли URL (слишком длинное обоснование в поле «Почему бумага») — при превышении лимита длины URL используйте короче текст или доработайте вызов через `doPost` на стороне клиента.
